# CEGTest 1.6 現行機能分析書

## 1. 概要

### 1.1 ツールの目的
CEGTest は**原因結果グラフ（Cause-Effect Graph）技法**によるテスト設計支援ツールである。
ソフトウェアテストの設計において、入力条件（原因）と出力結果（結果）の論理関係を視覚的に表現し、効率的にテスト条件を導出することを目的とする。

### 1.2 開発履歴
- 開発期間: 2010年1月 ～ 2013年8月
- 最終バージョン: 1.6 (2013年8月3日)
- 開発者: Masaki KASE (softest@nifty.com)

## 3. コア機能

### 3.1 ノード（NODE）

#### 3.1.1 ノードの種類
| 種類 | 説明 | 論理関係 |
|------|------|----------|
| 原因ノード | 入力条件を表す。グラフの左側に配置 | なし（葉ノード） |
| 中間ノード | 原因と結果を中継する論理的な中間状態 | AND/OR |
| 結果ノード | 出力結果を表す。グラフの右側に配置 | AND/OR |

#### 3.1.2 ノードのプロパティ
```javascript
NODE {
    text: string       // ノード名
    obs: boolean       // 観測可能性（テスト時に値を確認可能か）
    order: boolean     // 判定順序
    op: string         // 論理演算子 ("AND" | "OR" | "")
    seq: string        // 論理関係を表すカンマ区切り文字列
    lnum: number       // 論理展開式の数
    depth: number      // グラフ上の深さ
    top: number        // Y座標位置
    left: number       // X座標位置
    width: number      // 表示幅
    height: number     // 表示高
    effect: boolean    // 結果ノードか否か
    memo: string       // メモ
    zoom: number       // ズーム/分割状態（-1:通常）
}
```

#### 3.1.3 ノードの操作
- **配置**: メニューバーの[NODE]ボタンでモード切替、キャンバスクリックで配置
- **名前入力**: 配置時にテキストボックスが表示される
- **移動**: ドラッグ＆ドロップ
- **編集**: 右クリックコンテキストメニュー → プロパティ
- **論理関係設定**: 右クリック → [∧]または[∨]を設定
- **削除**: 右クリック → ノードの削除

### 3.2 論理関係（LINE）

#### 3.2.1 論理演算子
| 演算子 | 記号 | 意味 |
|--------|------|------|
| AND | ∧ | すべての入力が真のとき出力が真 |
| OR | ∨ | いずれかの入力が真のとき出力が真 |
| NOT | ～ | 否定（オレンジ色の線で表示） |

#### 3.2.2 ラインの描画
- [LINE]ボタンでモード切替
- 原因ノードをクリック（始点）
- 結果ノードをクリック（終点）
- NOT関係はダブルクリックで切り替え

### 3.3 制約（CONSTRAINT）

#### 3.3.1 制約の種類
| 制約 | 意味 | 例 |
|------|------|-----|
| ONE | 必ず1つだけが真 | ONE(A, B, C) → AかBかCのいずれか1つだけがT |
| EXCL | 同時に真にならない | EXCL(A, B) → AとBが同時にTにならない |
| INCL | 少なくとも1つは真 | INCL(A, B) → AかBの少なくとも1つがT |
| REQ | 要求制約（AならばB） | REQ(A→B) → AがTならばBもT |
| MASK | マスク制約 | MASK(A→B) → AがTならばBはM（don't care） |

#### 3.3.2 制約のプロパティ
```javascript
CONSTR {
    type: string       // "ONE" | "EXCL" | "INCL" | "REQ" | "MASK"
    seq: string        // 制約関係を表すカンマ区切り文字列
    top: number        // Y座標位置
    left: number       // X座標位置
    width: number      // 表示幅
    height: number     // 表示高
    memo: string       // メモ
    zoom: number       // ズーム状態
}
```

## 4. デシジョンテーブル生成

### 4.1 アルゴリズム概要

1. **論理式の展開**: 各結果ノードについて、AND/OR論理を展開
2. **テスト条件の選択**: 未網羅の論理式を優先的に選択
3. **制約の適用**: ONE/EXCL/INCL/REQ/MASK制約による値の決定
4. **演繹計算**: 論理関係から中間ノード・結果ノードの値を計算
5. **整合性検証**: 制約違反や論理矛盾がないか確認
6. **弱テスト条件の削除**: 他のテスト条件に包含される条件を除去

### 4.2 テスト条件の値
| 記号 | 意味 |
|------|------|
| T | ノードが真である |
| t | 論理上ノードが真でならざるを得ない（該当する真偽の組合せが論理式にない） |
| F | ノードが偽である |
| f | 論理上ノードが偽でならざるを得ない（該当する真偽の組合せが論理式にない） |
| M | MASK制約により真偽が決定できない |
| I | 原因側のノードが真偽不明のため真偽が決定できない |

### 4.3 カバレッジ表
- 各論理式がどのテスト条件で網羅されるかを表示
- 網羅されていない論理式は「テスト不可能」として表示

## 5. データ永続化

### 5.1 SessionStorage
- ブラウザを閉じるまでグラフデータを保持
- 次回アクセス時に自動復元

### 5.2 CSV形式（エクスポート/インポート）

#### エクスポートデータ構造
```
[ノード数],[制約数],
[ノード名1]
[ノード名2]
...

...（ノードごとに繰り返し）
[seq1]
[seq2]
...（論理関係）
[制約タイプ]
[制約seq]
...（制約ごとに繰り返し）
```

## 6. 高度な機能

### 6.1 ズームアウト（カプセル化）
- 中間ノードを選択し、その配下のノード群を非表示にする
- グラフの複雑性を低減し、大規模なグラフを扱いやすくする

### 6.2 分割（ズームイン）
- 中間ノードを選択し、その配下のみを表示する
- 大規模グラフの一部分に集中して作業可能

### 6.3 ノードの自動並べ替え
- ドラッグ終了時にノードの順序を自動調整
- 論理的な深さに基づいて配置

## 7. UI/UXの特徴

### 7.1 操作方法
- **ドラッグ＆ドロップ**: ノード・制約・テーブルの移動
- **右クリック**: コンテキストメニュー表示
- **ダブルクリック**: テーブル・カバレッジ表の非表示
- **ホバー**: ノード/制約にマウスを合わせると論理式を表示

### 7.2 視覚的フィードバック
- 選択中のツールはボタンが押下状態
- メッセージ領域に操作ガイドを表示
- エラーは赤色背景で表示

## 8. 既知の制限事項

### 8.1 機能的制限
- ノード数は最大100個
- 制約数は最大100個
- Undo機能は限定的（完全な履歴管理なし）
- グラフの画像エクスポートは未実装

### 8.2 ブラウザ互換性
- IEでは一部のグラフ表示に問題あり
- 推奨ブラウザ: Firefox、Chrome、Safari
